Terraform Basic Commands

  1. terraform init
  2. terraform validate
  3. terraform plan
  4. terraform apply
  5. terraform destroy

terraform init - used to initialize a working directory containing terraform config files - This is the first command that should be run after writing a new - terraform configuration downloads providers

terraform validate: validate the terraform configuration files in that respective directory to ensure they are syntactically valid and internally consistent

terraform validate: create an execution plan, terraform performs a refresh and determines what actions are necessary to achieve the desired state specified in configuration files

terraform apply: used to apply the changes required to reach the configuration, by default apply scans the current directory for the configuration and applies the changes appropriately

terraform destroy: used to destroy the terraform managed infrastructure this will ask for confirmation before destroying
